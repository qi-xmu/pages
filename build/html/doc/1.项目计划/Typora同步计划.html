Typora同步计划

  • typora自身提供对外接口的功能。提供了同步图片和文章接口。

  • Pandoc是一个好用的工具，可以将markdown文件转化成各种其他格式的文件。

  • 可以同步的地方，cnblogs，对象存储，gitee or github或者其他。

cnblogs: 博客园

目前不太清楚对外api的情况，不过可以确定它提供了对外接口，如果可以的话，可以实现一键发博的效果。对于图片资源和视频资源可以采用图床的方式。

对象存储

可以开发一个基于对象存储的静态网站，使用typora作为编辑器+pandoc将去转换成html，同时开发一个对应的软件进行管理。

Gitee/Github

这个只能作为同步，而且对文件的地址有要求，不过在修改README文件时比较方便。同时这个实现也比较简单。

git commit -am 'update README';git push

其他

也可以自己搭建一个前后端，实现o一键发blog的效果，不过需要自己拥有服务器，可以使用Docker大姐后端，再结合一些API框架搭建接口。

一起来做？

如果对这个项目感兴趣，可以一起做，目前我可能没有时间开始这个项目。
